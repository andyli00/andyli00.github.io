Work Term Report

Andy Li

Introduction

My name is Andy and I’m a 5th year Computer Science student at the University of Guelph. From January to September 2022 I worked as a co-op software developer at MANTECH Inc. This website is a summary and reflection on my time there.

About MANTECH Inc

MANTECH produces hardware and software used for water and soil analysis. This relates to computer science in that the hardware requires embedded software to function as well as standalone software which can communicate with the hardware. MANTECH has two software products; BOD Pro which is used to efficiently run 5-day biochemical oxygen demand (BOD5) analyses, and MATECH Pro which has centralized automation capabilities for a number of other analyses. I worked on MANTECH Pro during my co-op.

Goals

Going into the placement, I wasn’t really sure what to expect and didn’t particularly have any concrete goals as it was my first co-op and first experience programming in a professional setting. My goals were things like developing oral communication, teamwork, and technical skills. For my second set of learning goals halfway through my placement, I was able to come up with much more specific goals with the experience I had gained so far. My new goals were things like being better at giving people updates of what I’m doing, and improving the quality of my code to lower the amount of bugs that would be found by QC.

My Experience

As mentioned above, I worked on the MANTECH Pro software, which is capable of automating a number of chemical analyses through custom user-made scripting functionality. The software is written in C#, uses the Telerik .NET framework for UI, and interacts with a local sqlite database. It was my first experience using C# but the learning curve was fairly manageable since I had experience with Java in school. I was also familiar with using UI frameworks (JavaFX) and had taken CIS*3530 Databases previously.
After being familiarized with the codebase by making some UML diagrams and working on a few small features/bugs, I was given my first major project: database archiving and backups. The desire for archiving was because when users run scripts for analyses, a record of that run as well as records for whatever analyses are done are stored in the local database. Over time, the database becomes bloated with lots of old data that isn’t often needed. Archiving data lets it still be accessed if needed while also clearing up space in the local db and speeding things up. Eventually, the ability to backup databases was added onto this feature as there was a lot of shared functionality.
I started by planning out as much of the functionality as I could - I was asking various questions and discussing the feature during standups for a week or so. Thanks to this meticulous planning phase, work went fairly smooth when I started coding. I still ran into some issues, but not nearly as many as I would have without the planning I had done. Many of the issues I had were to do with the database; even though I had taken a database course which definitely helped significantly, I didn’t have real-world experience and so I learned a lot by working on this project.
I finished this project in a couple weeks, but since it wasn’t a major priority it wasn’t merged into the live product for a while. A few months later, I had some downtime between projects and decided to look back on this project as I now had more experience. Sure enough, there was lots to clean up. Most of the issues I found were just inefficient or messy code, but I did find a couple bugs which made me glad that I took another look at it and also made QC’s job a little easier.

Conclusion

Overall, my co-op at MANTECH was an extremely valuable first experience in professional software development. I met all of my learning goals and am satisfied with my performance. I couldn’t have asked for a better place to start my career in software development.
